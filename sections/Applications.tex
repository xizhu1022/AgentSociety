\section{Application Scope}
\label{sec:applications}

The ecosystem applies wherever agents hold heterogeneous expertise, quality is hard to verify unilaterally, and organizations operate with divergent interests. Across domains, agents switch fluidly between producer, consumer, and validator roles as context changes, and the dual loop's selection pressure gradually lifts the collective welfare of the agent society.

\begin{insightbox}{Representative Application Domains}
\begin{itemize}[leftmargin=1.5em, itemsep=4pt, topsep=2pt]
    \item \textbf{Software Engineering.} Software development requires deep specialization across languages, frameworks, and proprietary codebases. A React agent might write UI code for one task, call a Rust agent for backend services, and review a SQL agent's database design. Because validators can run tests or check code behavior directly, the PoI consensus here rests on computable evidence rather than subjective judgment, and weaker agents are naturally filtered out over time.

    \item \textbf{Scientific Research.} Scientific expertise is fragmented across subfields, and peer review today is slow and offers reviewers no real stake in their judgments. A chemistry agent might provide binding-energy data to a biology agent, request statistical help from a biostatistics agent, and review pharmacology claims from another agent. PoI turns peer review into a faster process where reviewers stake tokens on their assessments, and token flow across roles lifts the overall expertise of the scientific community.

    \item \textbf{Healthcare and Clinical Decision.} Clinical decisions are the highest-stakes setting of the trust crisis, where a wrong answer is measured in patient outcomes rather than dollars. A primary-care agent might consult a cardiology agent, and that cardiology agent might in turn consult a pharmacology agent or review another specialist's recommendation. Exaggerating one's competence costs real tokens here, and over time the ecosystem routes patient traffic to agents that consistently deliver reliable recommendations.

    \item \textbf{Legal Analysis.} Legal expertise differs sharply by jurisdiction, and consumers often cannot tell whether a legal analysis is correct until much later, if at all. A compliance agent might subscribe to specialists in EU, US, and Chinese law, with those specialists also validating each other's cross-border analyses. With no other reliable quality signal available, the PoI consensus is the primary check, and agents that consistently fail validation are gradually priced out, raising the overall quality of legal services available to non-experts.
\end{itemize}
\end{insightbox}

Each domain has its own practical challenges, such as bootstrapping the initial agent pool, latency constraints, and richer validation signals beyond a simple pass/fail verdict, which we leave to domain-specific follow-up work.
